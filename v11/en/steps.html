Using a Validate node to check a message against an XML schema

IBM Knowledge Center
Learn how to use the Validate node to check a message against an XML schema in a message flow driven by an XML message over HTTP.
  • Create
  • Prepare
  • Run

Import projects

When you click Import, one project will be created in your workspace: An application named ExampleValidate.

ExampleValidate provides a single message flow named Validate which receives data using an HTTPInput node. This node is configured to expect an XML message which conforms to ExampleModel.xsd although the HTTPInput node itself is not set up to validate the message immediately. The next node in the flow is a Validate node which checks the input XML against the XML schema. The message is sent down the Match or Failure terminal depending on whether validation passes or fails. Next the graphical mapping node generates a suitable reply which is returned using the HTTPReply node at the end of the flow.

Once imported using the link on the right, the provided project is ready to be deployed and tested. To deploy, you can use a provided BAR file, as described on the Prepare tab.

Create an integration node and deploy the message flow

Prepare the scenario by executing the following steps:

  1. Create an integration node by starting an App Connect Enterprise command console and executing the following commands:
    mqsicreatebroker TESTNODE
    mqsistart TESTNODE
    mqsicreateexecutiongroup TESTNODE -e server
  2. Back in the Toolkit, drag and drop deploy the provided BAR file named Validate.bar which you will find inside the ExampleValidate application.

Use the Flow Exerciser to test the tutorial scenario

  1. Open Validate.msgflow.
    • Click the Flow Exerciser icon to start testing the flow (when challenged you do not need to redeploy the flow again).
    • Click the Send Message icon .
    • Two saved messages are provided.
      The input message ExampleFAIL contains this data:
      This message will FAIL validation because it is not XML
      The input message ExamplePASS contains this data:
      <Message><Child>This message will PASS validation</Child></Message>
      Select one of these input messages and click Send and your message is sent to the HTTPInput node.
    • After you close the dialog, the path taken through the messageflow is highlighted. Try out both the input messages and note that the reply message reports the result of whether the validation passed or failed.

This tutorial has shown the Validate node check a a message against an XML schema.
